# Guide 04: Database Architecture

## Relational Design
- **Users**: Central model; roles linked via `UserRole`.
- **Content**: `Article`, `Video`, `ResearchPaper`, `Image` share a common `approvalStatus` workflow.
- **Engagement**: `Like` and `Comment` models support interaction across all multimedia types.
- **Auditing**: All content tracks `authorId` and `editorId` to maintain transparency.

## Maintenance
Use Prisma Studio to inspect or modify data directly:
```bash
npx prisma studio
```
This is essential for resolving data-level inconsistencies or verifying role assignments.
